# Interrupt Handling

This document traces the complete lifecycle of an interrupt — from the moment the CPU recognizes it through every assembly and C layer, back to the `iret` that returns to user space. Stack diagrams are given for each stage.

---

## 1. IDT Setup

The Interrupt Descriptor Table has 256 entries, one per vector (0x00–0xFF). It is populated in `boot_stage1()` → `setup_idt()`:

```c
_START static void setup_idt()
{
    unsigned long long *_idt   = GET_BOOT_ADDR(unsigned long long *, idt);
    unsigned long      *_stubs = GET_BOOT_ADDR(unsigned long *, intr_stubs);

    _stubs[SYSCALL_INT_NO] = (unsigned)syscall_handler; /* 0x80 → own stub */
    for (i = 0; i < IDT_SIZE; i++)
        _idt[i] = MAKE_INTR_GATE((void *)_stubs[i], 0); /* DPL=0 */
}
```

All 256 entries are created as **interrupt gates** (type 14) with **DPL 0** (kernel only). Interrupt gates cause the CPU to clear IF (disable interrupts) automatically on entry.

Two entries are updated later during kernel init:

| Vector | Gate type | DPL | Handler | When registered |
|--------|-----------|-----|---------|-----------------|
| `0x06` | interrupt | 3   | `handle_invalid_opcode` | `int.c` init |
| `0x0E` | interrupt | 0   | `pf_process` | `pagefault.c` init |
| `0x80` | **trap**  | 3   | `syscall_process` | `syscall.c` KERNEL_INIT 7 |
| `0xF0` | interrupt | 0   | `ipi_sched_handler` | `int.c` init |
| `0xF1` | interrupt | 0   | `ipi_tlb_handler` | `int.c` init |

The critical difference between gates:
- **Interrupt gate (type 14):** CPU clears IF on entry → hardware interrupts disabled.
- **Trap gate (type 15):** CPU does not touch IF → hardware interrupts remain enabled.

`int_register(vec, fn, is_trap, dpl)` (in `src/int/int.c`) lets subsystems update an entry after boot.

## 2. PIC / APIC Initialisation

### 8259 PIC (default)

Configured in `boot_stage1()` → `init_interrupt()`:

```
Master PIC (0x20/0x21):  IRQ 0–7  → vectors 0x20–0x27
Slave  PIC (0xA0/0xA1):  IRQ 8–15 → vectors 0x28–0x2F
All IRQ lines masked initially (OCW1 = 0xFF).
```

Individual drivers unmask their IRQ line when they register their handler.

### APIC

When `apic_init_bsp()` succeeds, `int_set_apic_mode()` sets `use_apic = 1`. From that point EOI is sent to the local APIC (`lapic_write(LAPIC_EOI, 0)`) instead of the 8259. IPI vectors (0xF0, 0xF1, 0xFF) always use APIC EOI.

## 3. Interrupt Entry Stubs (`src/int/int.S`)

One stub per vector is generated by a pair of macros. There are two variants depending on whether the CPU pushes an error code for that vector.

### Vectors WITHOUT a CPU error code — `zero` macro

```asm
#define zero        \
    pushl %ebp;     \   ← save caller's frame pointer
    pushl $0            ← synthetic error_code = 0
```

### Vectors WITH a CPU error code — `REAL` macro

When the CPU pushes an error code the stack at stub entry looks like:

```
ESP+0  error_code   ← pushed by CPU
ESP+4  EIP
ESP+8  CS
ESP+12 EFLAGS
...
```

The `REAL` macro shifts things so the layout is identical to the `zero` case:

```asm
#define REAL            \
    pushl (%esp);       \   ← copy EIP down one slot (make room for EBP)
    movl %ebp, 4(%esp)      ← write EBP into the now-free slot above error_code
```

After `REAL` the stack looks the same as after `zero`:

```
ESP+0  frame_pointer (EBP)
ESP+4  error_code
ESP+8  EIP
...
```

### STUB macro

```asm
#define STUB(NUMBER, TYPE)          \
    .text;                          \
    intr##NUMBER##_stub:            \
        TYPE;                       \   ← zero or REAL
        push $0x##NUMBER;           \   ← vec_no
        jmp intr_entry;             \
    .data;                          \
    .long intr##NUMBER##_stub;          ← stored in intr_stubs[]
```

Vectors that use `REAL`: `0x08`, `0x0A`, `0x0B`, `0x0D`, `0x0E`, `0x11`, `0x18`, `0x1A`, `0x1D`, `0x1E`.
All others use `zero`.

### Stack after the stub, before `intr_entry`

```
        ┌───────────────────────┐  ← low address (growing up)
ESP →   │ vec_no  (0xNN)        │  pushed last by STUB
        │ frame_pointer (EBP)   │  pushed by zero/REAL
        │ error_code            │  pushed by zero/REAL (0 or CPU value)
        │ EIP                   │  ─┐
        │ CS                    │   │ pushed by CPU on interrupt entry
        │ EFLAGS                │   │ (ESP/SS only added on ring change)
        │ ESP  (user)           │   │
        │ SS   (user)           │  ─┘
        └───────────────────────┘  ← high address
```

## 4. Common Entry Point — `intr_entry`

All stubs jump here. It saves the remaining registers and calls the C dispatcher.

```asm
intr_entry:
    push %ds
    push %es
    push %fs
    push %gs
    pushl %eax
    pushl %ecx
    pushl %edx
    pushl %ebx
    pushl $0          ← dummy ESP (pusha-style placeholder)
    pushl %ebp
    pushl %esi
    pushl %edi

    cld               ← clear direction flag

    pushl %edx
    mov $KERNEL_DATA_SELECTOR, %edx
    mov %edx, %ds     ← point DS/ES at kernel data segment
    mov %edx, %es
    popl %edx

    leal 56(%esp), %ebp   ← EBP → base of intr_frame (see below)

    pushl %esp            ← pass intr_frame * as argument
    call intr_handler
    addl $4, %esp
```

### Complete stack layout inside `intr_handler` (low → high)

```
        ┌─────────────────────────────────────────┐ ← ESP (lowest)
 +0     │ intr_frame *  (arg to intr_handler)      │ ← ESP when call executes
        ├─────────────────────────────────────────┤
        │         ── intr_frame ──                 │
 +4     │ EDI                                      │  ┐
 +8     │ ESI                                      │  │
 +12    │ EBP                                      │  │
 +16    │ 0  (dummy ESP)                           │  │
 +20    │ EBX                                      │  │ saved by intr_entry
 +24    │ EDX                                      │  │
 +28    │ ECX                                      │  │
 +32    │ EAX                                      │  │
 +36    │ GS  (16-bit + 16-bit pad)                │  │
 +40    │ FS  (16-bit + 16-bit pad)                │  │
 +44    │ ES  (16-bit + 16-bit pad)                │  │
 +48    │ DS  (16-bit + 16-bit pad)                │  ┘
        ├─────────────────────────────────────────┤
 +52    │ vec_no                                   │  ┐ saved by STUB
 +56    │ error_code                               │  │
 +60    │ frame_pointer (old EBP)                  │  ┘
        ├─────────────────────────────────────────┤
 +64    │ EIP   (return address in user/kernel)    │  ┐
 +68    │ CS    (16-bit + 16-bit pad)              │  │ pushed by CPU
 +72    │ EFLAGS                                   │  │
 +76    │ ESP   (user stack, only on ring change)  │  │
 +80    │ SS    (user stack, only on ring change)  │  ┘
        └─────────────────────────────────────────┘
```

`%ebp = ESP_at_intr_entry + 56` points to `vec_no`, which is the logical base of `struct intr_frame` as seen from C.

The C struct mirrors this layout exactly (`include/int/int.h`):

```c
typedef struct _intr_frame {
    unsigned int edi, esi, ebp, esp_dummy;   /* +0  .. +12  */
    unsigned int ebx, edx, ecx, eax;         /* +16 .. +28  */
    unsigned short gs, :16, fs, :16;          /* +32 .. +38  */
    unsigned short es, :16, ds, :16;          /* +40 .. +46  */
    unsigned int vec_no;                      /* +48         */
    unsigned int error_code;                  /* +52         */
    void        *frame_pointer;               /* +56         */
    void        (*eip)(void);                 /* +60         */
    unsigned short cs, :16;                   /* +64         */
    unsigned int eflags;                      /* +68         */
    void        *esp;                         /* +72         */
    unsigned short ss, :16;                   /* +76         */
} __attribute__((packed)) intr_frame;
```

## 5. C Dispatcher — `intr_handler`

```c
void intr_handler(intr_frame *frame)
{
    int external = frame->vec_no >= 0x20 && frame->vec_no < 0x30;
    int is_ipi   = (frame->vec_no == 0xF0 ||   /* IPI_VECTOR_SCHED    */
                    frame->vec_no == 0xF1 ||   /* IPI_VECTOR_TLB      */
                    frame->vec_no == 0xFF);    /* IPI_VECTOR_SPURIOUS */

    /* CPU exceptions (0x00–0x1F): re-enable interrupts.
     * They hold no IRQ line, need no EOI, and their handlers
     * (especially page fault) may need to sleep for I/O. */
    if (!external && !is_ipi)
        ENABLE_INTR();                  /* sti */

    int_callback fn = in_callbacks[frame->vec_no];
    if (fn)
        fn(frame);

    /* Send EOI so the controller accepts the next interrupt. */
    if (is_ipi || (external && use_apic))
        apic_eoi();
    else if (external)
        pic_end_of_interrupt(frame->vec_no);  /* OCW2 to 0x20 (+ 0xA0 for slave) */
}
```

### Interrupt enable/disable at each vector class

| Vector range | Gate type | CPU clears IF? | Dispatcher action | EOI |
|---|---|---|---|---|
| 0x00–0x1F (exceptions) | interrupt | yes → disabled | `sti` immediately | none |
| 0x20–0x2F (hardware IRQ, PIC) | interrupt | yes → disabled | stays disabled | PIC OCW2 |
| 0x20–0x2F (hardware IRQ, APIC) | interrupt | yes → disabled | stays disabled | LAPIC EOI |
| 0x80 (syscall) | **trap** | **no → enabled** | stays enabled | none |
| 0xF0–0xFF (IPI) | interrupt | yes → disabled | stays disabled | LAPIC EOI |

`iret` at the end of every path restores `EFLAGS` from the saved copy on the stack, which naturally restores IF to whatever it was in the interrupted context.

## 6. Interrupt Enable / Disable APIs

### Raw CPU flags

```c
ENABLE_INTR()          // asm volatile("sti")
DISABLE_INTR()         // asm volatile("cli")
GET_INTR_FLAG(f)       // pushfl; popl f
```

### Helper functions (`src/int/int.c`)

```c
unsigned int_is_intr_enabled(void);     // (EFLAGS & 0x200) != 0
unsigned int_intr_enable(void);         // sti; returns old state
unsigned int_intr_disable(void);        // cli; returns old state
void    int_intr_setlevel(unsigned en); // sti or cli based on en
```

The return value from `int_intr_disable` / `int_intr_enable` allows callers to restore the previous state (nested disable/enable):

```c
unsigned old = int_intr_disable();
/* ... critical section ... */
int_intr_setlevel(old);
```

### Spinlock and interrupt state

`spinlock_lock` disables the **scheduler** (not interrupts directly) via `sched_disable()` and saves the result in `lock->old_int`. `spinlock_unlock` restores it via `sched_set_level(lock->old_int)`.

`sched_disable` / `sched_enable` control an atomic flag (`scheduler_enabled`) that the scheduler checks before performing a context switch. This prevents involuntary preemption inside a critical section without requiring a full `cli`.

## 7. Return Path

After `intr_handler` returns, `intr_entry` continues:

```asm
    /* intr_handler returned */
    addl $4, %esp         ← pop the intr_frame * argument

    popl %edi
    popl %esi
    popl %ebp
    addl $4, %esp         ← skip dummy ESP
    popl %ebx
    popl %edx
    popl %ecx
    popl %eax
    pop  %gs
    pop  %fs
    pop  %es
    pop  %ds
    addl $12, %esp        ← discard vec_no, error_code, frame_pointer

    iret                  ← restores EIP, CS, EFLAGS (+ ESP, SS on ring change)
```

`iret` atomically restores `EFLAGS`, which re-enables interrupts (IF) if they were enabled in the interrupted context.

## 8. Syscall Path (INT 0x80)

The syscall entry has its own stub (`syscall_handler` in `int.S`) that bypasses `intr_entry` but builds the same `intr_frame` on the stack. It calls `intr_syscall_handler` instead of `intr_handler`.

Because the IDT entry is a **trap gate**, the CPU does **not** clear IF on entry. Hardware interrupts can fire freely during a syscall.

```
User space                 CPU (trap gate)           syscall_handler stub
──────────────             ────────────────           ─────────────────────
int $0x80       ───────►  push SS, ESP (ring chg)    push EBP (frame_ptr)
EAX = syscall#            push EFLAGS                push 0   (error_code)
EBX = arg1                push CS                    push $0x80 (vec_no)
ECX = arg2                push EIP                   push DS/ES/FS/GS
EDX = arg3                IF unchanged ──────────►   push EAX..EDI
ESI = arg4                                           call intr_syscall_handler
EDI = arg5
EBP = arg6
```

`intr_syscall_handler` looks up `in_callbacks[0x80]` and calls `syscall_process`:

```c
static void syscall_process(intr_frame *frame)
{
    syscall_fn fn = (syscall_fn)call_table[frame->eax]; /* EAX = number */
    int ret;

    if (!fn) ret = unhandled_syscall(frame->eax);
    else     ret = fn(frame->ebx, frame->ecx, frame->edx,
                      frame->esi, frame->edi, frame->ebp);

    frame->eax = ret;   /* return value written back into saved EAX */
    do_signal(frame);   /* deliver pending signals before returning */
}
```

Return is via the same register-restore + `iret` sequence, with `frame->eax` carrying the return value.

## 9. Page Fault Path (INT 0x0E)

Registered as an **interrupt gate** (IF cleared by CPU). The C dispatcher (`intr_handler`) re-enables interrupts because page fault handlers can block waiting for disk I/O.

```c
static void pf_process(intr_frame *frame)
{
    unsigned cr2;
    LOAD_CR2(cr2);          /* CR2 = faulting linear address */
    cr2 &= PAGE_SIZE_MASK;

    int oldint = sched_disable();   /* prevent involuntary task switch */

    if (!(frame->error_code & PF_MASK_P)) {
        /* Page not present → demand paging */
        pf_handle_page_invalid(cr2);
    } else if (frame->error_code & PF_MASK_RW) {
        /* Write to read-only page → CoW or permission upgrade */
        pf_handle_permission(cr2);
    } else {
        /* Unhandled: kill user task or kernel panic */
    }

    sched_set_level(oldint);
}
```

**Error code bits:**

| Bit | Name | 0 | 1 |
|-----|------|---|---|
| 0 | P | page not present | protection violation |
| 1 | W/R | read | write |
| 2 | U/S | kernel | user |

**Interrupt state during page fault:**

1. CPU entry: IF cleared (interrupt gate).
2. `intr_handler`: `ENABLE_INTR()` → interrupts re-enabled.
3. `pf_process`: `sched_disable()` → prevents preemption, but interrupts stay on.
4. Handler may call `task_sched()` (e.g., while waiting for a disk read to complete) — sleeping is safe because interrupts are on and the scheduler is only blocked for the faulting task.
5. Return: `sched_set_level(oldint)` → restore preemption.
6. `iret` → restore `EFLAGS` (IF).

## 10. Hardware IRQ Path (Vectors 0x20–0x2F)

```
IRQ fires on hardware line
        │
        ▼
CPU: saves EIP/CS/EFLAGS/(ESP/SS), clears IF (interrupt gate), vectors to stub
        │
        ▼
Stub: push EBP, push 0 (error_code), push vec_no, jmp intr_entry
        │
        ▼
intr_entry: saves DS/ES/FS/GS/EAX..EDI, switches DS/ES to kernel
        │
        ▼
intr_handler:
  external = true → interrupts stay DISABLED
  call fn(frame)   → driver ISR runs (e.g. keyboard, timer, e1000)
  pic_end_of_interrupt / apic_eoi  → EOI sent
        │
        ▼
intr_entry return: restores all registers
        │
        ▼
iret: restores EIP/CS/EFLAGS → IF restored from saved EFLAGS
```

Interrupts remain disabled for the entire ISR to avoid re-entrancy on the same IRQ line. The EOI is sent before `iret` so the PIC/APIC can accept the next interrupt on that line as soon as IF is restored.

---

## 11. IPI Path (Vectors 0xF0, 0xF1, 0xFF)

Inter-processor interrupts are sent via the local APIC ICR. Two are defined:

| Vector | Macro | Purpose |
|--------|-------|---------|
| `0xF0` | `IPI_VECTOR_SCHED` | Kick a CPU out of `hlt` so it re-checks the run queue |
| `0xF1` | `IPI_VECTOR_TLB` | TLB shootdown after a page table change |
| `0xFF` | `IPI_VECTOR_SPURIOUS` | APIC spurious interrupt (no-op) |

Treated identically to hardware IRQs: interrupts stay disabled, APIC EOI is sent before return.

## 12. Interrupt Enable/Disable Timeline (Full Picture)

```
User space (IF=1)
    │
    │  INT $0x80 (trap gate) ──── IF unchanged, still 1
    │  Hardware IRQ / exception ─ CPU clears IF → 0
    ▼
Stub runs (IF depends on gate type)
    │
    ▼
intr_entry runs (IF same as after stub)
    │
    ├─ exception (0x00–0x1F): intr_handler calls sti → IF=1
    │
    ├─ hardware IRQ (0x20–0x2F): IF stays 0 until iret
    │        spinlock inside ISR: sched_disable() + TAS (no extra cli)
    │        ISR completes → EOI → iret restores IF=1
    │
    ├─ syscall (0x80): IF=1 throughout
    │        spinlocks inside syscall: sched_disable() + TAS
    │        do_signal() → iret restores IF=1
    │
    └─ page fault (0x0E): CPU clears IF=0
             intr_handler sti → IF=1
             pf_process: sched_disable() (IF stays 1)
             handler may sleep (IF=1, scheduler off for this task)
             sched_set_level() → iret restores IF
```

## 13. Summary

| Event | IF at CPU entry | IF in handler | IF at iret |
|---|---|---|---|
| Exception (0x00–0x1F) | 0 (cleared by interrupt gate) | 1 (dispatcher does `sti`) | restored from EFLAGS |
| Page fault (0x0E) | 0 | 1 (dispatcher does `sti`) | restored |
| Hardware IRQ (0x20–0x2F) | 0 | 0 (stays disabled) | restored |
| Syscall (0x80) | 1 (trap gate, no change) | 1 | restored |
| IPI (0xF0–0xFF) | 0 | 0 | restored |
